# 校园轻集市 - 项目规划

---

## 一、页面清单

| 序号 | 页面名称 | 路由路径 | 是否需要登录 | 说明 |
|------|---------|---------|-------------|------|
| 1 | 首页/商品列表 | `/` | 否 | 商品浏览、分类筛选、搜索入口 |
| 2 | 商品详情 | `/product/:id` | 否 | 商品信息展示、图片轮播、联系卖家 |
| 3 | 发布商品 | `/publish` | 是 | 发布新商品表单（图片、描述、价格、分类） |
| 4 | 我的商品 | `/my-products` | 是 | 管理自己发布的商品（编辑、下架、删除） |
| 5 | 收藏列表 | `/favorites` | 是 | 查看收藏的商品 |
| 6 | 个人中心 | `/profile` | 是 | 用户信息、设置、我的收藏、我的发布入口 |
| 7 | 编辑个人信息 | `/profile/edit` | 是 | 修改头像、昵称、联系方式 |
| 8 | 登录页 | `/login` | 否 | 用户登录 |
| 9 | 注册页 | `/register` | 否 | 用户注册 |

## 二、功能模块

### 1. 用户模块
- **注册/登录**
  - 手机号/学号注册
  - 密码登录
  - 登录状态持久化（localStorage）
- **个人信息**
  - 查看/编辑头像、昵称、联系方式
  - 学生认证（预留）
- **权限控制**
  - 路由守卫拦截未登录用户
  - 登录后自动跳回原页面

### 2. 商品模块
- **商品浏览**
  - 商品列表展示（卡片式）
  - 分类筛选（教材、数码、生活用品、服饰、其他）
  - 排序（最新发布、价格低到高、价格高到低）
  - 下拉刷新、上拉加载更多
- **商品搜索**
  - 关键词搜索
  - 搜索历史记录
- **商品详情**
  - 商品图片轮播
  - 商品描述、价格、发布时间
  - 卖家信息展示
  - 联系卖家按钮
- **商品发布**
  - 多图片上传
  - 表单输入（标题、描述、价格、分类）
  - 表单验证
  - 发布成功提示
- **商品管理**
  - 我发布的商品列表
  - 商品状态（在售/已下架）
  - 编辑商品
  - 下架/重新上架
  - 删除商品

### 3. 交互模块
- **收藏功能**
  - 收藏/取消收藏商品
  - 收藏列表
- **联系卖家**
  - 展示卖家联系方式
  - 一键复制联系方式（预留）
- **浏览历史**
  - 记录浏览过的商品（本地存储）

### 4. 通用模块
- **布局组件**
  - 底部导航栏（TabBar：首页、发布、我的）
  - 顶部导航栏（标题、返回按钮）
- **通用组件**
  - 商品卡片组件（ProductCard）
  - 搜索栏组件（SearchBar）
  - 图片上传组件（ImageUploader）
  - 空状态组件（Empty）
  - 加载状态组件（Loading）
- **工具函数**
  - 时间格式化（多久之前）
  - 价格格式化
  - 本地存储封装
  - 请求封装

## 四、开发重点

### 1. 路由设计与导航守卫
- **重点原因**：路由是 SPA 的骨架，导航守卫是权限控制的核心
- **实现要点**：
  - 使用嵌套路由处理带导航栏的页面
  - meta 字段标记是否需要登录
  - beforeEach 全局守卫做权限拦截
  - 路由懒加载：`() => import('@/views/xxx.vue')`
  - 路由切换时的滚动行为处理

### 2. 组件复用与拆分
- **重点原因**：组件化是 Vue 的核心，好的组件拆分能大幅提升开发效率和可维护性
- **重点拆分组件**：
  - `ProductCard.vue` - 商品卡片（首页、我的商品、收藏列表复用）
  - `TabBar.vue` - 底部导航
  - `NavBar.vue` - 顶部导航
  - `SearchBar.vue` - 搜索栏
  - `ImageUploader.vue` - 图片上传
  - `Empty.vue` / `Loading.vue` - 状态组件
- **组件设计原则**：
  - props 向下传，events 向上传
  - 单一职责，不要做太多事
  - 预留插槽（slot）提高灵活性

### 3. 状态管理（Pinia）
- **重点原因**：用户登录状态、商品数据等需要跨组件共享
- **Store 划分**：
  - `useUserStore` - 用户信息、登录状态、token
  - `useProductStore` - 商品列表、缓存（可选）
  - `useFavoriteStore` - 收藏列表（可选）
- **注意事项**：
  - 持久化登录状态（localStorage）
  - 不要滥用 store，组件内部状态留在组件内
  - 使用 setup 语法风格的 Pinia

### 4. API 层封装
- **重点原因**：统一的请求封装能避免重复代码，便于统一处理错误和 loading
- **封装内容**：
  - 基于 fetch 或 axios 封装 request 函数
  - 请求拦截器：统一添加 token
  - 响应拦截器：统一处理错误（401 跳转登录）
  - API 模块化：`api/user.ts`、`api/product.ts`
  - Mock 数据支持（使用本地 JSON 或 MSW）
- **Mock 策略**：
  - 初期使用 Mock 数据，不依赖后端
  - 保留切换真实接口的能力

### 5. 移动端适配
- **重点原因**：校园集市主要使用场景是手机 H5
- **技术方案**：
  - viewport 设置：`<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">`
  - 使用 rem 或 vw 做单位
  - 或者直接使用 px，配合 PostCSS 转换（后续可加）
  - 触摸友好：按钮尺寸不小于 44px
  - 处理软键盘弹出问题

### 6. 表单处理
- **重点原因**：登录、注册、发布商品都涉及表单，表单体验直接影响产品质量
- **要点**：
  - 实时验证 rather than 提交时才验证
  - 友好的错误提示
  - 发布商品表单内容较多，注意分块或优化体验
  - 图片上传预览、删除功能

### 7. 用户体验细节
- **加载状态**：列表加载、按钮 loading
- **空状态**：没有数据时的友好提示
- **操作反馈**：Toast 提示（成功/失败）
- **页面返回**：正确的返回逻辑
- **图片优化**：懒加载、占位图
- **下拉刷新/上拉加载**：列表页必备
