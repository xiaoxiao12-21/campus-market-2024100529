## 五、项目规划（附）

### 页面清单

| 页面名称 | 路由路径 | 说明 |
|---------|---------|------|
| 首页/商品列表 | `/` | 商品浏览、分类筛选、搜索 |
| 商品详情 | `/product/:id` | 商品信息展示、联系卖家 |
| 发布商品 | `/publish` | 发布新商品（需登录） |
| 我的商品 | `/my-products` | 管理自己发布的商品（需登录） |
| 个人中心 | `/profile` | 用户信息、设置（需登录） |
| 登录页 | `/login` | 用户登录 |
| 注册页 | `/register` | 用户注册 |

### 功能模块
1. **用户模块**：注册/登录、个人信息管理、学生认证
2. **商品模块**：列表、筛选、搜索、详情、发布、管理
3. **交互模块**：收藏、联系卖家、浏览历史
4. **通用模块**：底部导航、搜索栏、状态提示、图片懒加载

### 开发顺序
1. Day 1-2：基础框架搭建（路由、页面骨架、布局组件、Pinia 结构）
2. Day 3-4：用户模块（登录注册、状态管理、路由守卫、个人中心）
3. Day 5-6：商品核心功能（列表、详情、Mock 数据、API 封装）
4. Day 7：商品发布与管理（发布表单、我的商品、编辑下架）
5. Day 8：交互与优化（收藏、联系、状态处理、样式优化）

### 开发重点
- 路由设计与导航守卫
- 组件复用与拆分
- Pinia 状态管理
- API 层封装
- 表单处理与验证
- 移动端用户体验
